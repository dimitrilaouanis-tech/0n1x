The problem

Agents claim. Nobody checks.

As AI agents transact autonomously over x402, AP2, MCP and ERC-8004, there is no way to confirm a merchant, price, or claim is real before value moves. UC Berkeley showed all eight leading agent benchmarks can be reward-hacked to near-perfect scores without completing a single task. The gap between what agents say and what they do is a measured, systemic risk — and the payment rails deliberately left the verification seat empty.

The empty seat

The one thing nobody else signs.

The stack already signs who an agent is (identity, KYA, passports) and enforces policy. Payment rails — x402, AP2, Visa TAP, Mastercard — verify payment integrity. None of them verify whether the merchant, price, or fact the agent is about to act on is actually true. That is the seat 0n1x occupies, posting to the deliberately-empty ERC-8004 Validation Registry rather than competing with the rails.

Security · the first discipline

AI security is not a feature here. It is the product.

0n1x exists because autonomous agents move value with no immune system. We built the immune system — a Tier-1 agent-safety firewall at the payment chokepoint, a permission-audit layer, and hardened cryptography — and we measured the industry with it.

FIREWALL

Verify at the chokepoint

Signed PROCEED / REVIEW / HOLD before an agent settles: recipient firewall, pre-sign preflight (catches unlimited-approve), EIP-712 permit-drain detection, token risk, agent liveness — fused into one clearance.

2,227

We scanned the agent economy

Live posture scan of 2,227 public agents: 99.1% expose no permission scope, 29.3% run plain HTTP, and 386 of 387 value-moving agents have no spend cap. Only 2 were flag-clean. Signed posture records issued for every one.

PERM

Permission & mandate audit

The third leg of trust: is the action an agent is about to take actually inside its granted mandate? Signed IN-SCOPE / OUT-OF-SCOPE facts (PERM_v0 → AP2 mandates), plus signed permission grants.

HARDENED

Battle-tested cryptography

Ed25519 + JCS canonical signing with pinned-key verification (key-substitution attacks closed), prompt-injection guard with invisible-unicode detection, challenge-response nonces on public challenges. Attack, don’t trust.

UNTRUSTED

Agent text is data, not commands

Every inbound agent message is treated as untrusted input. Outbound is public-safe by policy. The protocol assumes adversaries, because the agent economy already has them.

SIGNED

Nothing asserted, everything provable

Every security verdict is a signed fact you can verify offline against our published key — not a black-box score. Neutral by design: 0n1x earns nothing from what it grades.
