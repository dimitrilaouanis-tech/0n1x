The problem

Agents claim. Nobody checks.

As AI agents transact autonomously over x402, AP2, MCP and ERC-8004, there is no reliable way to confirm a merchant, price, or claim is real before value moves — or to tell an agent's stated behavior from its actual behavior. In April 2026, UC Berkeley (RDI) showed all eight leading agent benchmarks — SWE-bench, WebArena, OSWorld, GAIA among them — could be reward-hacked to near-perfect scores without completing a single task. The gap between what agents say and what they do is a measured, systemic risk.

01 · Sign

Ed25519 receipts of what happened

Independently verifiable signed receipts of agent actions and ground-truth facts — merchant, price, outcome — anchored to a live Merkle truth-root.

02 · Verify

Verify before you pay

Before settling, an agent calls 0n1x and gets a signed fact it can check against our published key. No trust in us required — only in the math.

03 · Neutral

Earns nothing from what it grades

The structural moat: no conflict of interest, published methodology, and a compounding signed track record. Code is copyable; a reputation is not.

The empty seat

The one thing nobody else signs

The stack already signs who an agent is (identity registries, KYA, passports) and enforces policy. Payment rails — x402, AP2, Visa TAP, Mastercard — verify payment integrity. None of them verify whether the merchant, price, or fact the agent is about to act on is actually true. That's the seat 0n1x occupies, posting to the deliberately-empty ERC-8004 Validation Registry rather than competing with the rails.

For agents

How agents reach it

0n1x is a paid MCP server over x402 (USDC on Base mainnet) — no API key, no signup; the wallet is the identity. Tools include signed verify-before-pay attestations, onyx_aeo_score (the only AEO score with published weights, 3+ runs and a 95% CI, signed), and onyx_erc8004_lookup — signed on-chain reads of the ERC-8004 registries.
