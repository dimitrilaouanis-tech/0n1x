ProtocolMCP · July 2026

MCP's next spec goes stateless — and the agent tooling layer gets a scaling unlock

The Model Context Protocol release candidate dated 2026-07-28 makes MCP easier to run, reason about and extend in agentic systems. The headline change: MCP becomes stateless at the protocol layer — removing the sticky-session assumption that kept agent tooling from scaling horizontally.

Why it matters for 0n1x: MCP is the agent-to-tools layer we already expose over x402 on Base. A stateless core is precisely what lets a metered verify endpoint fan out without session affinity — the difference between a demo and a rail.

Sources: State of Agentic AI Standards 2026 — DEV · MCP Is Growing Up — AAIF

PaymentsAP2 · 2026

Google hands the Agent Payments Protocol to the FIDO Alliance

Ownership of AP2 moved to FIDO for community governance — mirroring A2A's move to the Linux Foundation. v0.2 ships Human-Not-Present support for autonomous, pre-authorized transactions, and layers on top of A2A and MCP.

For 0n1x: AP2's three signed Mandates — Intent, Cart, Payment — are the exact shape of verify-before-you-pay. Our x402 rail already separates the grader key from the payee: the property AP2 formalizes.

Sources: ap2-protocol.org · Google Cloud
ProtocolA2A · Apr 2026

A2A crosses 150+ organizations and lands in production clouds

The Agent2Agent standard now counts 150+ participating organizations and 22,000+ GitHub stars, with production deployments inside Azure AI Foundry and Amazon Bedrock AgentCore. Governance sits with the Linux Foundation.

For 0n1x: Our agents speak A2A natively — signed Agent Cards, identity verified before action, settlement over x402. The horizontal layer is now table stakes, not a bet.

Sources: Stellagent · Zuplo
ModelsOpenAI · Jul 2026

GPT-5.6 ships as three models built for agents

GPT-5.6 arrives as Sol (frontier), Terra (~5.5-level intelligence at half the cost) and Luna (small, fast), plus an Ultra mode with a Max reasoning level and heavier sub-agent use. New API surface: programmatic tool calling, multi-agent orchestration, prompt-cache breakpoints.

For 0n1x: Cheap frontier reasoning plus native orchestration is the substrate a live agent network runs on. The moat was never the model — it's signed proof-of-execution on top of it.

Sources: BuildFastWithAI · ThursdAI
ModelsAnthropic · Jun–Jul 2026

Anthropic makes agents cheaper to run with Claude Sonnet 5

Sonnet 5 launched as a lower-cost way to run agents, while Claude Cowork expanded to web and mobile — letting agents run multi-step tasks in the background across devices. Claude Code shipped a broad stability pass: subagent streaming, better permissions, faster rendering.

For 0n1x: Falling cost-per-agent-run is what makes an always-on fleet economical. A network compounds only when running an agent 24/7 stops being expensive.

Sources: TechCrunch · Anthropic Newsroom
The stackAgentic commerce · 2026

The 2026 agent stack has converged: MCP + A2A + AP2

The ecosystem settled on three complementary layers — MCP for agent-to-tools (vertical), A2A for agent-to-agent (horizontal), AP2 for payments — each governed by an independent foundation. The plumbing debate is over.

For 0n1x: We sit across all three. The question the standards don't answer is trust — is this agent, and this fact, real? That unanswered layer is the whole company.

Sources: NiteAgent · AgenticPlug
