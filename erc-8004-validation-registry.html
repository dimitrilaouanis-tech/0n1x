The ERC-8004 Validation Registry, explained

ERC-8004 ("Trustless Agents") deploys three singleton registries at fixed addresses across EVM chains — an Identity Registry, a Reputation Registry, and a Validation Registry. The Validation Registry is where independent parties post outcome and fact attestations about agents on-chain — the standard's open write-seat for verifiable, third-party agent facts.

The three registries

The Identity Registry records that an agent exists (one NFT per registered agent). The Reputation Registry records feedback and scores. The Validation Registry records attestations — statements from an independent validator about what an agent did or whether something is true. On Base mainnet (eip155:8453) the Identity and Reputation singletons are live at 0x8004A169…a432 and 0x8004BAa1…9b63.

Why the Validation seat matters

Most agent-trust projects write to Identity or Reputation — they score the agent. Almost nobody occupies the Validation seat with facts about the merchant, price, or outcome an agent transacts against. That is the deliberately-empty socket in the standard: independent, signed, on-chain fact attestation.

How 0n1x uses it

0n1x targets the Validation Registry with Ed25519-signed fact attestations — verify-before-pay clearances and Proof of Agent Execution receipts — anchored so any agent can check them without trusting 0n1x itself. You can read the registries live and signed via the onyx_erc8004_lookup tool, which returns decoded registry metadata and the current agent count.
