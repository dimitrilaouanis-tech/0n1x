The question

Will a specific external domain's TLS certificate rotate (its notBefore advance / reissue) within a 7-day window? Per-domain, not "will some cert appear in a firehose" — the latter has a base rate near 1 and is trivial in the opposite direction. Per-domain rotation is the skill question.

How you'll recompute it (on 07-20)

  1. Both snapshots (07-13 and 07-20) — the per-domain cert notBefore for all 114 domains — will be attached to this page as JSON.
  2. Rotation = notBefore(07-20) > notBefore(07-13) for a given domain.
  3. Measured 7-day rate = rotations / 114. Persistence = 1 − rate.
  4. Admit iff persistence < 0.95 and the rate sits in the pre-registered band. Anyone can rerun step 2–4 from the attached snapshots and get the same answer.

Result: VOID — unresolvable

Verdict: VOID. Not admitted, not rejected — unresolvable. The 07-13 per-domain baseline (each domain's notBefore) was never persisted; only the aggregate survived. A measurement that does not exist cannot be created after the fact — time moves forward — so the named 07-13 → 07-20 pair cannot be honestly diffed. Reconstructing or remembering a baseline would be the exact fabrication this page exists to prevent. Discovered 2026-07-19 during pre-resolution checks, published on schedule.

The search (evidence of the empty result): checked the earn-eligibility ledger (aggregate only: n=114, observed_rotations=6, no domain list), the churn state (_rank_churn_latest.json — leaderboard, no domains), _predict_rank.json (counts, resolvable_domains is a number not a list), and the 63 MB commitments log (_predict_commitments.jsonl — no notBefore baseline). A VOID published on schedule, before we knew it was coming, is a stronger record than an admit or a reject: it proves the process contains failures it did not have to confess. That is the point of building a machine to catch yourself.

Root-cause fix (banked, so it can't recur)

Same disease as the coin-flip grader and the stale daemon: the system kept the conclusion and discarded the checkable inputs. New standing policy — every pre-registration writes its raw baseline to a signed, published file at registration time, not just the aggregate. Recompute is only real if the ground truth it recomputes from was persisted.
